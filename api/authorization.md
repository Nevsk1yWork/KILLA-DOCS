# Авторизация

{% hint style="info" %}
* **URL для запросов:** `https://proxy.killa.cc/api/v1`
* **Авторизация:** для каждого запроса передавайте токен в заголовке `Authorization: Bearer <token>`.
{% endhint %}

<details>

<summary><strong>Памятка</strong></summary>

* Токен = пароль. Не передавайте его в URL, не логируйте и не отправляйте в чаты.
* Храните токен в переменных окружения / секрет-хранилищах.
* Если токен утёк — **сразу сбросьте** его в боте. Старый токен перестаёт работать сразу.

</details>

### 1) Где взять токен

В Telegram-боте: **Профиль → API → скопируйте токен**.

### 2) Заголовки

| Name                                                | Type   | Description                                           |
| --------------------------------------------------- | ------ | ----------------------------------------------------- |
| Authorization<mark style="color:$danger;">\*</mark> | string | `Bearer <token>`                                      |
| Content-Type                                        | string | `application/json` (только для запросов с JSON-телом) |

### 3) Сброс токена

В Telegram-боте: **Профиль → API → Сбросить токен**.

* Старый токен становится невалидным сразу.
* Новый токен нужно обновить во всех интеграциях.

### 4) Пример запроса

```bash
curl -s https://proxy.killa.cc/api/v1/balance \
  -H "Authorization: Bearer YOUR_TOKEN"
```
